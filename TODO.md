# Пошаговый план реализации микросервиса Оркестратора ИИ-агентов (FastAPI + LangGraph + Temporal)

Этот документ представляет собой детализированный TODO-лист для реализации микросервиса **Оркестратора** (Control Plane) в рамках мультиагентной системы. Внимание: данный репозиторий предназначен исключительно для сервиса Оркестратора. Реестр агентов (Registry) и изолированный Проигрыватель (Player) предполагаются как внешние контракты/микросервисы (или интегрированные в виде вызовов к LangGraph в рамках диспетчеризации).

## Этап 3: Алгоритмы Макро-Оркестрации (Разрешение DAG)

Ядро Control Plane: построение и анализ графа зависимостей агентов.

- [x] **3.1. Алгоритм обратной трассировки (Backward Tracing)**
  - Написать логику рекурсивного или итеративного поиска зависимостей целевого агента (через `AgentDiscoveryClient`), пока не найдутся корневые агенты (`dependencies == []`).
- [x] **3.2. Обнаружение циклов (Cycle Detection)**
  - Реализовать защиту от зацикливания зависимостей (выброс ошибки Deadlock Detected).
- [x] **3.3. Топологическая сортировка (Topological Sort)**
  - Использовать алгоритм Кана (Kahn's algorithm) для преобразования DAG в последовательность волн (Execution Waves), где агенты внутри одной волны не зависят друг от друга.

## Этап 4: Разработка Temporal Workflows и Activities

Координация асинхронного, отказоустойчивого выполнения плана.

- [x] **4.1. Разработка Activities**
  - `DiscoverRootAgentActivity`: вызов семантического поиска.
  - `BuildExecutionPlanActivity`: вызов алгоритма обратной трассировки и топологической сортировки.
  - `ExecuteAgentActivity`: вызов микро-исполнения.
    - *Вариант А (External Player)*: Отправка HTTP-запроса `POST /api/v1/player/execute` во внешний микросервис.
    - *Вариант Б (Embedded Player/LangGraph)*: Запуск изолированного графа LangGraph прямо внутри воркера на основе `AgentManifest`.
- [x] **4.2. Разработка Orchestrator Workflow**
  - Написать Workflow, который последовательно:
    1. Находит стартового агента.
    2. Строит DAG-план.
    3. Выполняет `ExecuteAgentActivity` параллельно для агентов из одной волны (`asyncio.gather` в Temporal).
    4. Собирает ответы агентов в State Store.
    5. Формирует `input_context` для следующих агентов на основе выходов предыдущих.
    6. Возвращает итоговый результат.
